HAIL - HUMANS ARE ILL LIVED

HAIL or Humans Are Ill Lived is my second Game Jam game for Ludum Dare 33. I made this game with my friend Harvey and we decided for the theme “You are the Monster” we went with a more “artistique” approach. In the game there are 4 different monsters which you fight. You only have one attack move but you also have the ability to move left and right and jump. The goal here is to dodge and then attack at the right time.

When making the game we encountered quite a variety of challenges. First of all, neither of us had used source control before so we didn’t bother using that. Instead we decided to go with a plugin to Eclipse called Saros and it allow for real time development together. So this made it quite easy for the both of us to develop the small game using only a handful of files without having constant merging issues. As we were both programmers we often overwritten each others code so Saros happen to be our light and saviour in this case.

Another challenge we encountered was the artifical intelegence part of the monsters. We wanted a good challenge and a little less predictiable then most boss rush games. We both believe that we have made something very interesting and creates for enough game play for a 2 day game development cycle.

I created the small engine we used and also all the graphics you see on the screen. I’m not much of an artist but with a restricted colour pallete I manage to get quite a lot out of a little. I hope to do this again in the future because there is some interesting ideas in this game aswell as the asthetic that accompanies it.

3D SOFTWARE RASTERIZER

For my second semester I was given a unit that required us to implement mathematics specific to 3d graphics. In my assignment I had created a Ray Tracer, Rasterizer, Curves, Fractals and Primitives. This is by far my most proud project that I completed as part of my university.

Implementing a Rasterizer was quite a journey and I learnt a lot about 3D graphics along the way. I had previously had a look at OpenGL so performing the same transformations required to get points on the screen was not to hard. The hard part came when I needed to implement a trangle rasterizer. I used an algorithm which split the triangle into 2 (flat-bottom and flat-top triangles), and that seemed to do the trick. From there I could use texturing, lighting and colours to create full 3d rendered objects on the scene.

NEURAL NETWORKS AND GENETIC ALGORITHMS

At my University I was tasked with finding a neural network or genetic algorithm solution to the traditional A* path-finding method. Whilst researching this area I have come across an older paper on the use of both artificial neural network and genetic algorithms in tandem. This was achieved by lining up all the weights into a chromosome and then performing a genetic algorithm evolution on that chromosome.

Implementing this was quite a task. I decided to use no preexisting libraries and relied solely on the CPU to perform the training and fitness tests. The end result was a working yet wonky result. It wouldn’t find a very good path but it did find a path, and once trained this can be much faster then A*. This still needs some work to be used effectively in video games.

The training method for this neural network is a genetic algorithm. This is achieved by taking all the weights in order by index and putting them into a single array, after this the so familiar genetic algorithm can be applied and then a fitness test to calculate a fitness of the offspring. This works very well however you will need to calculate a new set of weights for each map for it to actually work and find a path at all.

As you can see the layout of the neural network is quite simple and has only 1 hidden layer. A back propagation training method might be more effective, however I did not have enough time to implement this. I hope to move forward with trying neural networks on models and more advanced A.I. for characters / CPU’s.

OBJECT ORIENTED PROGRAMMING

For my Object Oriented Programming assignment I decided to focus on my programming techniques, so alot of the code is actuallly more engine code than game code. I used SFML as a base and elaborated from there by implementing Game, Input, Scene and GameObject managers to create a more coherant engine.

The game featured is a game called Power Shot and in the game the goal is to plant turrets on the ground and then fire them. Each barrel will shoot one bullet and will only last for a short peroid of time. The goal is to strategically place the turrets so you can remove as many of the numbered blocks as possible. The number on the blocks signify how many times the blocks can withstand a hit. Once all the blocks on the screen are gone the area is conquered and you can move onto the next level.

I believe this game would fair better on mobile platforms due to portrait screen and also the mouse being the only input. Maybe at a later date this can be adapted to iOS and Android.
